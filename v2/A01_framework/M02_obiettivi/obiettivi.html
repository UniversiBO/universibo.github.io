Obiettivi e requisiti

Costruire un framework per applicazioni web con le seguenti caratteristiche:

  • Sistema di configurazione flessibile non code-based
  • Facilmente estendibile attraverso la definizione di comandi applicativi relativi allo spazio concettuale delle specifiche applicazioni, definizione dei modalità per la loro invocazione
  • Separazione della rappresentazione dell'output dalla logica applicativa permettendo diverse viste
  • Integrazione di un accesso alla persistenza dei dati su database relazionale con un layer di astrazione indipendente dalla particolare implementazione del database
  • Sistema di error handling flessibile e semplice da utilizzare per sopperire alla carenza del linguaggio
  • Sistema di logging delle operazioni sensibili
  • Toolbox di strumenti utili in grado di includere altre classi open-source di uso comune
  • Utilizzo e di una pura Object Oriented methodology, limitando il più possibile free-standing code e variabili globali per ragioni di sicurezza e pulizia delle applicazioni
  • Fornire supporto ed "instradare" verso lo sviluppo di applicazioni 3-tier basate sul pattern MVC.

Il framework deve permettere di soddisfare gli obiettivi e requesiti del portale per il quale è nato... si veda la sezione obiettivi e requisiti generale
