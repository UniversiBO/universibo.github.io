Il modello di sviluppo Open Source

UniversiBO è un progetto Open Source che ha l'intento di sviluppare un software libero, con il contributo della comunità degli studenti. E' necessario che quest'ultimi abbiamo un ruolo fondamentale nel definire le linee di sviluppo dei servizi in quanto fruitori di questi.
Questo modello di sviluppo implica delle modalità di collaborazione e lavoro (diffondere le release presto e spesso, delegare ad altri tutto il possibile, essere aperti alle modifiche ed integrazione degli altri sviluppatori, ecc... ) a riguardo consiglio di leggere l'articolo introduttovo La cattedrale e il bazaar - Eric S. Test Driven Development (...o quasi)

Il Test Driven Development (TDD) è una tecnica di sviluppo nata all'interno dell'Extreme Programming (XP) che prevede di scrivere il codice a partire dai test.
Non vogliamo utilizzare questa tecnica portandola al suo estremo, ma scrivere solo una suite completa di test dell'applicazione utilizzando PHPUnit.
Alcune regole importanti da seguire sono:
- Non fare il commit del nuovo codice che avete scritto prima di averne scritto anche dei test!!
- Eventualmente se state lavorando con un'altra persona a fianco mettetevi daccordo per dividervi reciprocamente la scrittura dei test e quella del codice ed eseguendole in parallelo.
- Se viene segnalato un bug, scrivete prima un test che lo mette in risalto poi correggetelo.

  • home page di PHPUnit: http://pear.php.net/package-info.php?pacid=38

APACHE+PHP+POSTGRESQL for DUMMIES (NB windows only)

Obbiettivo: installare una copia locale di universibo sul mio bel piccì di casa.

Software usato (necessario):

  • apache (for win) 1.3.28 (lo trovi qui)
  • php 4.3.2 (lo trovi qui)
  • postgresql 7.3.x (scaricatelo insieme a cygwin oppure separatamente da qui)
  • cygwin 1.5.0-1 (qui) NB il file setup serve sia per scaricare i pacchetti necessari da internet risolvendo eventuali problemi di dipendenze che per installare il tutto
  • cygipc 1.14-1 (qui ) ATTENZIONE! al momento in cui scrivo l'ultima versione (la 2.0) non funziona
  • phppgadmin 3.0 (qui)

Attenti i link al software indicato possono essere vecchi o non funzionanti, vi consiglio di cercare con un motore di ricerca le ultime versioni.

 

1° PASSO: installazione di apache

Avviate l'eseguibile apache_1.3.28-win32-x86-no_src.msi e seguite le istruzioni di installazione. E' molto imprtante scegliere "Run as service for all users" ed effettuare installazione completa. (nel mio caso come directory di installazione ho scelto d:\apache)


2° PASSO: installazione di php e settaggio di php.ini & httpd.conf

Scompattate il file zip in un cartella ad uopo (nel mio caso c:\php). A questo punto copiate php.ini-dist nella directory di windows, rinominandolo però php.ini
Aprite php.ini con blocco note e tramite il comando Trova individuate la riga ;extension=php_pgsql.dll e togliete il ; davanti.
Cercate sempre allo stesso modo la riga extension_dir = e settatelo come c:\php\extensions
Cercate sempre allo stesso modo la riga pgsql.max_persistent = e settate a 3.
Cercate sempre allo stesso modo la riga pgsql.max_links = e settate a 4.
Cercate sempre allo stesso modo la riga session.save_path = e settate con il nome della directory da usare per i file temporanei ( D:\Documents and Settings\fab\Impostazioni locali\Temp nel mio caso).
Nella cartella Php dobbiamo cercare un file chiamato "php4ts.dll" (Attenzione: se non lo vedete vuol dire che dovete selezionare, nel menù Visualizza>Opzioni Cartella>Visualizza, il checkbox relativo a Mostra tutti i file). Il php4ts.dll è fondamentale per il funzionamento del modulo Php. Una volta trovato dobbiamo copiarlo nella directory C:\Windows\System (C:\Winnt\System32 per i possessori di Windows NT/2000/XP).
Adesso dobbiamo modificare il file httpd.conf di Apache (X:\apache\conf\httpd.conf) .
Ecco i passaggi da effettuare:
Cercate la riga ThreadsPerChild e settate a 30.
Sempre con l'apposita funzione del blocco note cercate la riga #LoadModule unique_id_module modules/mod_unique_id.so e, subito sotto, aggiungiamo quanto segue: LoadModule php4_module c:/programmi/php/sapi/php4apache.dll ( se necessario modificate il percorso che porta al file php4apache.dll ).
Cercate la riga AddModule mod_setenvif.c e sotto aggiungiamo AddModule mod_php4.c
Cercate ServerName e lo impostate come 127.0.0.1
Cercate DocumentRoot e lo impostate come "D:/apache/WWW" NB WWW è la cartella root di localhost dove verranno interpretati i file php
Cercate la riga con <Directory e impostatela a "D:/apache/WWW" NB deve essere lo stesso nome dato per DocumentRoot
Cercate AddType application/x-tar .tgz e sotto aggiungete:

AddType application/x-httpd-php .php
AddType application/x-httpd-php .php3

Per completare l'opera cercate la riga <IfModule mod_dir.c> e trasformate DirectoryIndex index.html in DirectoryIndex index.php index.html index.htm index.php3 index.phtml
In poche parole abbiamo aggiunto, separati da spazi, i nomi delle pagine che Apache deve cercare in automatico nella directory principale.
Adesso nella directory adibita a localhost (WWW nel mio caso) create un file phpinfo.php contenente la seguente riga:

<?php phpinfo(); ?>

Aprite il vostro browser e nella barra degli indirizzi digitate http://localhost
Cliccate su phpinfo e se tutto è andato bene compare una schermata con le info sulla versione di php installata e sulle opzioni impostate.


3° PASSO: scaricare e installare cygwin

Avviate setup.exe . Una volta scelta la modalità di installazione, i pacchetti da scaricare e installare, e quindi da selezionare, sono:
+ categoria base: tutti
+ categoria database: postgresql
+ categoria Interpreters:gawk,libiconv,libintl1,libintl2,libncurses5,libncurses6,libncurses7,libpcre,termcap
+ categoria libs:crypt
+ categoria shells:ash,bash,sh-utils
+ categoria system:rebase,procps
+ categoria text:texinfo
+ categoria utils:bzip2,rebase
NB mi raccomando effettuate l'installazione sempre per tutti gli utenti.
Complimenti avete installato cygwin!! adesso dovreste averne l'icona sul desktop.
Copiate il file compresso ( .bz2 ) di cygipc nella directory dove avete installato cygwin.
Fate doppio clic sull'icona di cygwin. Si apre la finestra dos con la shell di cygwin. Al prompt date i seguenti comandi:

cd /
bzip2 -d cygipc-1.13.2.tar.bz2 (a seconda della versione)
tar -xvf cygipc-1.13.2.tar

Sempre nella shell di cygwin dare i seguenti comandi:

[lancia cygipc]
$ ipc-daemon &
NB dovrebbe comparire una risposta del tipo [1] xyzc dove xyzc è il pid del processo; memorizzate questo numero che ci servirà per arrestare il daemon
[lancia postgres]
$ initdb -D /usr/share/postgresql/data/
$ postmaster -D /usr/share/postgresql/data/ -i &
[lancia l'interfaccia da riga di comando postgres]
$ psql -d template1 (NB serve solo per crearti l'utente la prima volta)
create user postgres password yes superuser (NB al posto di yes potete mettere la password che più vi aggrada)
\q

NB lasciate la finestra di cygwin aperta!!


4° PASSO: installare phppgadmin

Unzippate il file nella directory root di localhost (d:\apache\WWW nel mio caso)
Aprite il file config.inc.php (si trova nella directory conf) nell'editor che preferite e modificate la riga $conf['servers'][0]['host'] = in $conf['servers'][0]['host'] = '127.0.0.1';
BENISSIMO!!
Arrivati a questo punto avete finito le installazioni dei programmi necessari!! Resta solo qualche piccola configurazione da fare...


5° PASSO: installazione del database di prova di universibo

Loggatevi a phppgadmin:
-aprite il browser
-digitare localhost nella barra degli indirizzi
-cliccate su phppgadmin
-nella finestra di login (appena comparsa) digitare username: postgres password: yes
Bene a questo punto è comparsa la schermata di gestione di phppgadmin.
Nel frame di sinistra cliccate su PostgreSQL.
Nel frame principale cliccate su crea database.
Come nome per il database scrivete universibo.
Cliccate sul nome del database appena creato.
Nel frame centrale in alto a destra cliccate su sql.
Scompattate il file universibo_db_schema_test.zip e aprite il file dump_test.
Copiate tutto il contenuto e incollate nel riquadro sql di phppgadmin.
Cliccate su vai.
Bene! Avete installato il vostro db di prova!!
Adesso scompattate universibo_1.3.4.zip nella cartella root di localhost (d:\apache\WWW nel mio caso)
Ora dovete configurare i seguenti files:

  • htmls/forum/config.php
    settate così:
    $dbhost = '127.0.0.1';
    $dbname = 'universibo';
    $dbuser = 'postgres';
    $dbpasswd = 'yes';


  • includes/include/settings.inc.php
    settate così:
    $pg_dbuser = 'postgres';
    $pg_dbpassword = 'yes';

Avvertenze:

Ogni volta che volete accedere ad universibo non scordatevi di avviare postgre tramite cygwin!!
Per evitare di dare sempre tutti i comandi a mano può essere comodo creare due file di script (semplici file di testo coi comandi da dare), per esempio start e stop.

[Testo da scrivere nel file start]
ipc-daemon &
pg_ctl start -D /usr/share/postgresql/data/ -o -i

[Testo da scrivere nel file stop]
pg_ctl stop -D /usr/share/postgresql/data/ -m fast
kill -9 $(procps -C ipc-daemon -o pid=)
